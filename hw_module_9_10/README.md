# Завдання 1: Порівняльний аналіз алгоритмів видачі решти

Реалізовано та досліджено два підходи до розв'язання задачі для касового апарату: **Жадібний алгоритм** (`find_coins_greedy`) та алгоритм **Динамічного програмування** (`find_min_coins`).

Заміри середнього часу одного виконання демонструють різницю між алгоритмами:
**Поведінка Жадібного алгоритму:**
$O(K)$: константна (для фіксованого набору монет), $O(1)$: пам'ять лише під фінальний словник
Для різних сум час виконання залишається **константним** - оскільки алгоритм замість послідовного перебору використовує операції ділення націло та взяття залишку (швидкість залежить лише від кількості доступних номіналів у касі).

**Поведінка Динамічного програмування:**
$O(N \cdot K)$: лінійна відносно суми, $O(N)$: потребує два масиви розміром `amount + 1`
При збільшенні суми решти час виконання **росте лінійно** - оскільки внутрішні цикли змушені прораховувати оптимальні комбінації для кожного проміжного числа від `1` до `N`.
⚠️ **Важливий момент:** при дуже великих значеннях решти динамічне програмування починає споживати занадто багато пам'яті під створені масиви, що може призвести до уповільнення всієї системи.

   * $N$ (це сума решти)
   * $K$ (кількість доступних номіналів монет)

## Висновки
   * **Жадібний алгоритм** варто використовувати як основний алгоритм касового апарату за замовчуванням. Він забезпечує миттєву видачу правильної решти без навантаження на процесор (лише у випадку, якщо доступний повний набір номіналів).
   * **Динамічне програмування** можна використовувати як додатковий варіант (підключати лише у випадках, якщо в касі закінчилися певні номінали монет, через що жадібний алгоритм не може гарантувати мінімальну кількість монет для покупця).

# Завдання 2: Обчислення визначеного інтеграла методом Монте-Карло

Для реалізації цього завдання й обчислення визначеного інтегралу методом Монте Карло, було вибрано тригонометричну функцію $f(x) = \sin(x) + 1$ на проміжку від $a = 0$ до $b = \pi$ .
Для оцінки методу результати було порівняно з чисельним методом з бібліотеки SciPy (`scipy.integrate.quad`).

При генерації $N = 100,000$ випадкових точок у прямокутнику $[0, \pi] \times [0, 2]$ було отримано такі дані:
*   **Метод Монте-Карло:** `5.143716`
*   **`quad` (SciPy):** `5.141593`
*   **Абсолютна похибка методу Монте-Карло:** $\approx 0.002123$

## Висновки
Алгоритм MC успішно впорався з обчисленням площі під кривою синусоїди (результат збігається до другого знака після коми, із вбудованим інструментом). Сама ж точність залежить від величини вибірки ($N$) (похибка складає менше 0.05%).
