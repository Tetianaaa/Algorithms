### Insertion Sort: 
На рандомних та зворотних масивах час зростає квадратично $O(N^2)$, а збільшення розміру масиву, відповідно призводить до збільшення часу сортування. Вже на відсортованому масиві із лінійною складністю відпрацював швидше за Сортування злиття $O(N)$.

### Merge Sort
Час виконання залишається майже ідентичним для різних типів масивів $O(N \log N)$. Краще працює на великих масивах, але створює додаткові витрати на малих масивах (у порівнянні з Insertion).

### Timsort
Timsort показав найкращий результат у всіх тестах. На відсортованих даних він працює за лінійний час $O(N)$. На випадкових даних він значно швидший за ручне сортування злиттям.
